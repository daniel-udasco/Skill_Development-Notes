checkbox - commonly used in forms, that may have more than one answer.

for (attribute) - another way to associate an input element's text with the element itself. To use this, you can nest the text within a label element and add a for attribute with the same value as the input element's id attribute.
^ Ex. 
<input id="breakfast" type="radio" name="meal" value="breakfast."> <label for="breakfast">Breakfast</label>

*whats the difference between
A.
<input id="loving" type="checkbox" name="personality"> <label for="loving">Loving</label>
B.
<label><input id="loving" type="checkbox" name="personality"> Loving</label>
^ Ans:
Option A offers more flexibility in layout, especially if the label needs to be positioned separately from the checkbox.
Option B is more concise and ensures that the entire label (including text) is clickable, providing a more intuitive user experience.

While either value is optional, it's best practice to include it with any checkboxes/radio buttons on the page.

checked (attribute) - make a checkbox checked or radio button selected by default.
^ Ex.
<!--There's no need to set a value to the checked attribute.-->
<input checked type="radio" name="meal" value="breakfast"> Breakfast

HTML elements have an opening and closing tag with content in between. The following is the basic syntax of HTML:
^ Ex. <openingTag>content</closingTag>

h1 (element) - main heading of a webpage.

The h1 through h6 heading elements are used to signify the importance of content below them. The lower the number, the higher the importance, so h2 elements have less importance than h1 elements. Only use one h1 element per page.

p {paragraph} (element) - used to create a paragraph of text on websites.

Comments in HTML - starts with (<!--) and ends with (-->).

HTML5 has some elements that identify different content areas. These elements make your HTML easier to read and help with Search Engine Optimization (SEO) and accessibility.

main (element) - represent the main content of the body of an HTML document. Content inside should be unique and should not be repeated to the document.

Try to indent nested elements to make it easier to read for you and others.

img (element) - add images to your website.

void (element) - element without a closing tag.
^ Ex. <img, meta, etc.> (no </img> or </meta>)

HTML attributes - special words used inside the opening tag of an element to control the element's behavior.

src {source} (attribute) in img - specifies the image's URL (where the image is located).
^ Ex.
      <img src="https://cdn.freecodecamp.org/curriculum/cat-photo-app/relaxing-cat.jpg">

alt (attribute) - text is used for screen readers to improve accessibility and is displayed if the image fails to load.
^ Ex. <img src="cat.jpg" alt="A cat">

All img elements should have an alt attribute.

a {anchor} (element) - used to link to another page.
^ Ex. <a href="https://www.freecodecamp.org"></a>

link text - must be placed between the opening and closing tags of an anchor (a) element.
^ Ex. click here to go to freeCodeCamp.org
